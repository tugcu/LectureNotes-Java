Instance Methods for the Course Class

  • Static methods belong to the class definition, not to the individual objects.
    • If it is expected to operate on a specific object, that object should be passed as a parameter to the static method (beats the purpose of having objects)
    • A static method can be invoked:
      • within a method of that class, directly by its name (e.g., sort(myArray))
      • from outside the class, through the name of that class, not the object variable's name (e.g., Utils.sort(myArray) or the familiar method call Math.pow(x,y))
  • Instance methods, on the other hand, are specific to the individual objects.
    • No need to mention the object name as a parameter since it is obvious.
    • An instance method can be invoked:
      • within a method of that class, directly by its name (e.g., registerCourse(courseName))
      • from outside the class, through the name of the object (e.g., student.registerCourse(courseName))
