Private Fields and Methods

  • To restrict access to a field or method of a class from other classes, you should define it as private.
  • A private field or method is visible only within that class. Therefore, objects of other classes cannot read or modify the values of private fields, or invoke private methods.
  • The intuition behind making a field private is to ensure that the client code does not perform an uncontrolled access that may cause problems later. For example, you might want to ensure that the value of the private field cannot be set to a negative value by mistake.
  • Similarly, you may want to ensure that a method is not invoked from other classes since it may cause unexpected side effects.
  • Information hiding in this manner is an important issue in software engineering as well as security.
