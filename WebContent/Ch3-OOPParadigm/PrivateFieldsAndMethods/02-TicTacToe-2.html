Designing the TicTacToe Game

  • From the object-oriented point of view:
    1. The game is actually an interaction between two objects: player1 and player 2.
    2. These two objects make use of another object: a gameboard composed of 9 squares.
    3. An orchestrater that initiates the game, keeps track of whose turn it is, whether the game is over, who the winner is, etc.

  • So, we have three different types of objects, i.e., three different classes:
    1. Player class: Set the respective player object with its name and mark.
    2. Board class: Manage the GUI (all graphics operations, mouse clicking, displaying info, etc.)
    3. Game class: Initialize the board and the players, run the game through several rounds, check if the game is over or not, etc.
  • We also need another class, Main, to trigger the execution of the whole project, i.e., provide the main method.
  • We will organize related classes in separate packages, framework and runnable.
