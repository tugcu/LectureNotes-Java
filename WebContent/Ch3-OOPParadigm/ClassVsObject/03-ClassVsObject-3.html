Classes vs. Objects

  • Note that typically a class is not a program on its own
    • Each class is a separate type defined in a separate file
    • The program is composed of several classes (several files)
    • Each class is a template for objects of that specific type
    • There will be one orchestrating class (with the main method) that initiates execution
      • Typically you create instances of one or more classes in main. It continues as a ripple effect.
