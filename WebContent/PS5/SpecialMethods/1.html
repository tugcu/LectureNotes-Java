equals()

  • All objects have both identity (the object's location in memory) and state (the object's data). The == operator always compares identity. The default implementation of equals() compares identity as well.
  • But if you override equals(), it should compare state, not identity.
  • You must be very careful while implementing equals() and consider many corner cases:
    • Objects to be compared may not be of the same class
    • They may be null
    • Fields to be compared must have already overriden equals()
    • ...
  • For the time being, I am not covering stuff related to hashCode(). We will come to this later on.
  • For more examples, see http://www.javapractices.com/topic/TopicAction.do?Id=17
