What is a LinkedList?

  • A data type in which each data item has a link to its successor. Typically, a LinkedList is a collection of Node's. Both data and the link to the successor is stored in the Node.
  • The good thing is: It is a dynamic data structure, that is, it can be easily updated (insertion, deletion) while the program is running. So, stop worrying about whether there is an empty position in the underlying structure or not.
  • The bad thing is: You use extra memory for pointers and the access to the elements are usually sequential, i.e., you visit a lot of nodes before reaching your target.
  • Our implementation will be generic: At the end of today's PS, we will be able to store any data type we want in our data structure.
