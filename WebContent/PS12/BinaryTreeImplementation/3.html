  • Well, it won't actually work. The reason is this magical sentence:
    In Java, object references are pass-by value.
    Being more clear, if method A calls method B and method B initializes a Node instance to a new instance using new, the pointer in the callee (method A) is not updated, it maintains the former value.
  • So, the problem is the same as the following: 
		
public class Main {

	public static void main(String[] args) {
		Node<Integer> r = new Node<Integer>(10);
		System.out.println(r.data); // prints 10
		update(r);
		System.out.println(r.data); // prints 10, not 20
	}
	
	public static void update(Node<Integer> node) {
		node = new Node<Integer>(20);
	}
}
