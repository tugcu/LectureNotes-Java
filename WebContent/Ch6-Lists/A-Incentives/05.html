List As a Collection

  • A list is a linear configuration of nodes that can grow or shrink during execution
    • Thus, the first disadvantage is circumvented
  • A list can be implemented in two ways:
    • Using an array as the underlying technology (e.g. ArrayList)
      • Still subject to the second disadvantage (low performance for random insert/delete)
    • Using links to setup relationships between disjoint items, thus a linked list
      • Circumvents the second disadvantage
