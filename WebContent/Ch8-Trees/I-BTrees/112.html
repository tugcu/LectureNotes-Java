An M-Way Tree of Order 3

  • The figure on the next page shows the same data as the previous figure, stored in an M-way tree of order 3. In this example M=3 and h=2, so the tree can support 9 leaves, although it contains only 8.
  • One way to look at the reduced path length with increasing M is that the number of nodes to be visited in searching for a leaf is smaller for large M.
  • We will see that when data is stored on the disk, each node visited requires a disk access, so reducing the nodes visited is essential.
